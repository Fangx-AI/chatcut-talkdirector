# ChatCut TalkDirector README Design

## Goal

Turn the public GitHub front page into a high-impact but immediately understandable product page for a director Skill dedicated to already-shot talking-head videos.

The page must communicate one idea within ten seconds:

> ChatCut TalkDirector reads spoken content and decides when to keep the speaker, add motion graphics, create supporting visuals, change composition, or leave the footage clean.

## Naming

- Product name: **ChatCut TalkDirector**
- Chinese name: **ChatCut 口播导演**
- Repository name: `chatcut-talkdirector`
- Chinese positioning line: **读懂口播，导演画面。**
- English positioning line: **Read the words. Direct the visuals.**
- Skill identifier remains `chatcut-talking-head-visual-director` for compatibility.

## Visual Direction

Use an editorial post-production language rather than generic AI neon styling.

- Palette: ink black, warm ivory, signal green, and a small REC red accent.
- Typography: large editorial display title paired with compact mono production labels.
- Motifs: talking-head frame, caption safe area, waveform, transcript anchors, and sparse Visual Beat markers.
- Avoid: fake ChatCut interfaces, cinematic equipment, dense HUD decoration, blue-purple gradients, glow, and badge walls.

The content stays simple while the presentation feels like a premium campaign and production-system page.

## Hero Asset

Create a 21:9 masthead containing:

- a professionally photographed talking-head presenter as the first visual signal;
- a clear camera frame and caption-safe region;
- sparse waveform and Visual Beat markers;
- the product name and positioning line typeset outside the generated raster layer;
- enough contrast for GitHub display at desktop and narrow widths.

Use Image 2 to generate the presenter/source plate without embedded text. Compose exact typography and interface marks deterministically so all words remain correct.

## Supporting Visual Assets

1. **Directing flow:** transcript to director judgment to MG, generated visual, split/full-screen, or clean footage.
2. **Speaker composition atlas:** original frame, split-screen, adaptive PiP, and temporary full-screen.
3. **Quality Gate comparison:** overloaded effects versus sparse high-value Beats.
4. **Visual Beat Map example:** a compact, truthful example of the deliverable without claiming a real ChatCut render.

All supporting assets use the same palette, spacing, label system, and visual hierarchy as the hero.

## README Structure

1. Centered hero masthead.
2. Product name, two positioning lines, compact factual badges, and anchor navigation.
3. The problem: already-shot talking-head footage is visually repetitive.
4. The directing model: the visual follows meaning rather than adding effects everywhere.
5. The five-way routing system with supporting graphics.
6. A concise Visual Beat Map example.
7. Confirmation and Quality Gate behavior.
8. Install and invocation.
9. Repository map, validation evidence, and current limitations.

The README is Chinese-first with concise English labels where they strengthen the brand.

## Claims And Boundaries

- Do not claim a real ChatCut render, generated media result, or timeline execution.
- State that planning is confirmed before credit-consuming actions.
- State that one representative Beat is verified before expansion.
- Keep the existing 9/9 scenario result and 12-reference count factual.
- Do not imply that lower-right PiP is the default.
