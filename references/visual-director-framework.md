# 口播视觉导演框架

## 导演目标

让视觉服务口播的理解、记忆、节奏、情绪和转场。不要把“做得更多”当成“做得更好”。人物表达足够强时，保持干净是正式结论。

## 读取输入

优先获取：

- 完整 transcript 和时间戳
- 目标平台、画幅和视频时长
- 人物在代表性画面中的位置、大小和手势范围
- 字幕、Logo、产品界面和已有文字的位置
- 用户品牌、受众和希望保留的视觉风格

已有项目上下文足够时直接判断。只有一个缺失信息会改变视觉语言、构图或执行风险时，最多追问一个问题。没有精确时间戳时可以标注“约”并使用文案锚点，不能伪造精确帧位。

## 决策顺序

1. 保留原口播声音和已完成的内容剪辑，不默认重写文案。
2. 识别 hook、观点、步骤、数据、对比、案例、场景、反转、结论和 CTA。
3. 只把能改善理解、记忆、节奏、情绪或转场的句子列为候选 Beat。
4. 为整条视频选择一个主视觉系统。
5. 为每个候选 Beat 选择 MG、生成画面、真实 B-roll、全屏视觉、PiP、分屏或保持干净。
6. 决定人物是否露出以及放在哪里。
7. 运行 Quality Gate，删除或降级不值得做的 Beat。
8. 输出 Visual Beat Map，标出代表性 Beat，等待确认。

不要为了避免 overlay 构图困难而随意切全屏，也不要为了持续露出人物而给所有补画面叠加 PiP。

## 视觉密度

- 30-60 秒默认 3-8 个 Beat。
- 60-120 秒默认 5-12 个 Beat，但连续三个重视觉 Beat 后应安排留白。
- 信息密度低、人物表演强或画面拥挤时可以少于默认值，甚至 0 个。
- 同一句只保留一个主视觉任务；需要同时解释和强调时，先解释，再用轻量文字落点。

## 每个视觉必须承担一个主要工作

- 解释：让抽象概念、流程或关系变得可见。
- 记忆：突出关键词、数字、定义、结论或金句。
- 节奏：打断长时间不变的画面，但不制造无意义噪音。
- 情绪：放大反转、冲突、期待或收束。
- 转场：明确章节变化并自然回到人物。

无法写出主要工作时，选择保持干净。

## 代表性 Beat

从通过 Quality Gate 的 Beat 中选择一个最能同时验证视觉语言、文字层级、人物构图和动画节奏的点。优先选择中等风险且可快速判断好坏的 Beat，不优先选择最昂贵或最复杂的生成镜头。

## 确认边界

导演阶段只输出方案。用户确认后先执行代表性 Beat；用户看到实际效果并同意后，才扩展到其余 Beat。任何会触发生成额度或修改时间线的动作都属于执行阶段。
