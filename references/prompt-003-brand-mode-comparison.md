# Prompt 003: 品牌双模式能力对比

用一个品牌图标串联两种产品模式：第一阶段并排解释能力差异，第二阶段把信息收束为一句结果对比。适合普通版与专业版、聊天模式与办公模式、免费版与付费版，以及“给建议”与“直接交付”等双模式讲解。

[观看已验证演示视频](../assets/verified-prompts/prompt-003-brand-mode-comparison.mp4)

## 用户直接使用的 Prompt

```text
在 [时间段] 制作一段横屏全屏品牌双模式对比动效。

使用纯黑背景，顶部居中放置 [品牌] 的可验证官方图标。图标作为整段的品牌锚点，贯穿两个信息阶段，不要在中间转场时单独淡出或重新出现。

第一阶段并排展示两种模式：
- 左侧标题「[模式 A]」，只保留一项核心能力「[能力 A]」。
- 右侧标题「[模式 B]」，按照口播顺序依次出现 2-4 项能力：[能力 B1]、[能力 B2]、[能力 B3]、[能力 B4]。

左侧使用中性灰弱化，右侧使用一个高亮强调色。标题使用大号高对比中文，能力项使用可快速扫读的短词组。右侧能力逐项落下，每次只新增一个视觉重点；最后一项最强，但不要使用持续辉光、粒子、玻璃卡片或小号英文装饰。

第二阶段让第一阶段的标题和能力块一起退场，品牌图标保持稳定，然后把画面收束为「[能力 A] → [最终结果 B]」，下方补充一句「[总结句]」。不要让图标单独淡出；如果整段结尾需要退场，图标必须与最终文字同步。

音效使用轻、短、清脆的卡点：标题入场一次空气感短扫，右侧能力项使用逐级增强的短促弹点，最终结论使用一次干净短扫。避免低沉 boom、机械拖尾和连续 whoosh，口播始终优先。

保持原视频画幅和时长，不生成或仿造真实产品界面。先展示第一阶段稳定帧、能力递进帧、中间转场帧、最终结论帧和导出成片的实际第一帧让我确认；独立导出时，第一帧不得露出上一镜。
```

只需要替换时间段、品牌、两种模式、能力项、最终结果和总结句。能力项必须来自用户原文、逐字稿或已经确认的产品事实，不由模型擅自扩写。

## 观看任务

- 第一阶段回答“两种模式分别能做什么”。
- 第二阶段回答“它们最根本的区别是什么”。
- 品牌图标负责身份连续性，不承担转场动画。
- 画面的震撼感来自字号、对比、节奏和信息收束，不来自装饰数量。

不适合多于两种模式、需要逐项阅读长段说明、必须操作真实产品界面，或没有任何可验证品牌素材的片段。

## 官方品牌素材

按以下顺序获取品牌图标，并记录来源：

1. 项目内已有的官方素材
2. 官方 Brand、Press 或 Media Kit
3. 官方网站 SVG 或 PNG
4. 官方应用商店图标
5. 官方 favicon

不得生成、重绘或风格化真实品牌 Logo。使用前检查实际画布、透明留白、深浅色版本和适合的背景。找不到可验证素材时停止执行并请求用户提供。

## 信息结构

1. 左侧必须只有一个核心能力。若模式 A 需要解释多个能力，本 Prompt 不适用，应改用普通对比表或拆成两个 Beat。
2. 右侧默认 2-4 项能力，按口播顺序出现。超过 4 项时拆分，不压缩字号。
3. 两侧标题先建立对比，再出现能力项。左侧不需要为了“对称”而补充虚构条目。
4. 最终结论必须是第一阶段的语义收束，不引入新卖点。
5. 页面不放编号、英文眉题、技术标签或无意义小字，除非它们本身就是用户要表达的内容。

## 构图与视觉规则

- 默认横屏 16:9、纯黑背景、全屏 MG；不保留人物 PiP。
- 品牌图标位于顶部中轴，建议宽度为画面宽度的 11%-14%，四周保留足够黑色呼吸区。
- 第一阶段左右视觉重量约为 45:55；模式 A 弱化，模式 B 强调。
- 标题使用巨型高对比中文色块或纯文字，能力项字号必须能在手机横屏预览中快速扫读。
- 强调色只承担模式 B、最后能力和最终结果，不再增加第二个高饱和色。
- 背景保持纯黑；不使用渐变、网格、粒子、玻璃拟态、持续辉光或装饰性英文。
- 图标在两个阶段之间保持位置、尺寸和不透明度稳定。禁止“主体文字淡出，图标提前消失”或“图标消失后又重新淡入”。

## 默认节奏

- 建立品牌与双标题：总时长的 0%-18%。
- 第一项能力：约 18%-32%。
- 模式 B 后续能力：按照口播或均匀短间隔逐项落下，约覆盖 32%-62%。
- 第一阶段整体退场：约 62%-78%，标题和能力块使用同一淡出窗口。
- 最终结论进入并稳定：约 75%-100%。
- 品牌图标从片段开始贯穿到结束；只有用户明确要求整段退场时，才与最终文字共用同一退场窗口。

实际执行优先使用逐字稿词点和真实口播节奏，不机械套用百分比。

## 音效规则

- 标题：一次轻空气感短扫。
- 每个新增能力项：一个约 8-12 帧的清脆短弹点；可以逐级提高 1-2 dB，但不盖住口播。
- 最终结论：一次干净、短促的转场扫声。
- 禁止低沉 boom、低频 riser、厚重机械卡扣、长拖尾和连续扫声。
- 没有口播混音环境时宁可更轻，不用音效补偿画面节奏。

## 可编辑属性

- 品牌官方图标、缩放、位置、圆角和背景适配
- 模式 A / B 标题
- 模式 A 的唯一核心能力
- 模式 B 的 2-4 项能力及出现顺序
- 最终左右结论与总结句
- 字体、文字色、中性灰、强调色和纯黑背景
- 各阶段时长、能力项词点、整体淡出窗口和最终结论进入时间
- 标题短扫、能力弹点、结论短扫的素材和音量

## 验证

执行后至少检查：

- 官方品牌图标的独立预览与来源
- 片段第一帧和第一阶段稳定帧
- 模式 B 每个能力项的新增状态
- 第一阶段淡出前、淡出中和完全退场帧
- 最终结论进入、稳定和结尾帧
- 图标在两个阶段之间的位置、尺寸和不透明度连续性
- 导出文件的实际第 1 帧，确认没有上一镜泄露
- 卡点音效是否轻、脆、短，且口播优先

向用户展示实际结果并获得第二次确认后，才能复用到同一视频的其他对比 Beat。
