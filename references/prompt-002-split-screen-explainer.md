# Prompt 002: 双栏讲解

左侧讲结论，右侧展示长文本证据流。适用于 Prompt、代码、报告、合同、论文或产品需求等内容的横屏讲解。

## 用户直接使用的 Prompt

```text
在 [时间段] 制作一段横屏双栏信息动效。

左侧作为主视觉，显示标题「[标题]」，并将以下要点按照叙述顺序逐条浮现：
1. [要点一]
2. [要点二]
3. [要点三]
4. [要点四]

每次只突出当前要点。当前项使用强调色和状态标记，已经出现的内容降低亮度但继续保留，尚未出现的内容隐藏。最后进入全部完成状态。

右侧作为辅助信息区，完整放入以下长文本，并让文字持续、匀速、缓慢地由下向上滚动：
[粘贴长文本]

右侧文字用于体现内容的完整性和信息量，不要求在视频结束前展示全文，不要为了滚完全文而加快速度。左侧必须是主视觉，右侧宽度不得超过画面的 45%。保持原视频时长和画幅，不添加无关装饰，并先展示开始、中段和结束关键帧让我确认。
```

只需替换时间段、标题、要点和长文本。没有现成口播画面时，可直接制作成独立无声视频，之后再搭配配音。

## 观看任务

- 左侧负责让观众记住结论和结构。
- 右侧负责证明内容确实来自一份完整材料，不承担逐字阅读任务。
- 两栏不能争夺同一层级的注意力；当前左侧要点始终是唯一焦点。

不适合只有一句短文案、必须逐字读完全文，或需要同时展示可操作产品界面的片段。

## 导演规则

1. 左侧占画面约 55%-62%，右侧占 38%-45%；右侧不得成为面积或亮度上的主视觉。
2. 默认使用 3-5 个要点。超过 6 个时拆成两个 Beat，不压缩字号硬塞。
3. 当前项最亮；历史项降低亮度但保留；未出现项隐藏；结尾进入全部完成态。
4. 右侧保留 Markdown 标题、列表、代码缩进和段落层级，不把长文本压成一整块。
5. 右侧滚动速度以舒适为先，默认每秒移动右栏可视高度的 10%-15%。
6. 不要求滚完全文。禁止为了展示完整内容而加快滚动、缩小到不可读字号或扩大右栏。
7. 默认做成全屏解释 Beat。保留原口播音频；除非人物露出对理解有必要且存在真实安全区，否则不添加人物 PiP。
8. 没有原片时可以输出独立无声视频，不生成虚构人物或无关 B-roll。

## 默认节奏

- 开场建立标题与双栏关系：总时长的 8%。
- 各要点平均分配剩余主要时段，使用短距离滑入和透明度变化。
- 当前项切换时，右侧保持连续匀速滚动，不突然跳页或加速。
- 最终完成态：总时长的 8%。

如果用户给出逐字稿或明确语义锚点，以真实口播节奏为准，不机械平均分配。

## 可编辑属性

- 时间段与总时长
- 标题、要点数量、要点标题与说明
- 右侧完整长文本
- 左右栏比例
- 滚动速度与滚动方向
- 字体、主文字色、次文字色、强调色和背景色
- 当前项、历史项和完成态样式

## 验证

至少检查：

- 开场双栏构图
- 每一个要点的当前状态
- 历史项与当前项是否能明显区分
- 右侧长文本在中段和结尾是否连续、无裁切、无突然加速
- 右侧是否超过画面 45% 或抢夺左侧焦点
- 结尾完成态与退场
