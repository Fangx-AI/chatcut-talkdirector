# Visual Beat Map

## 输出顺序

1. 整体导演判断
2. Visual Beat Map
3. 代表性 Beat
4. 保持干净的片段
5. 高风险确认项
6. 用户确认后的执行顺序

按时间排序 Beat。缺少精确时间时使用原文锚点和“约”时间，不伪造时间码。

## 整体导演判断

```markdown
# 口播视觉导演方案

- 视频目标：
- 平台与画幅：
- 已知画面条件：人物、字幕、Logo、产品和空白区
- 导演假设：没有则写“无”
- 主视觉语言：色彩、字体、形状、运动和纹理
- 建议视觉密度：每 30-60 秒 X 个 Beat
- 全局保护区：
- 代表性 Beat：Beat XX，原因
```

## 单个 Beat 模板

```markdown
## Beat 01：简短名称

- 文案锚点：原文关键句
- 时间位置：精确或约时间
- 视觉目的：解释 / 记忆 / 节奏 / 情绪 / 转场
- 推荐手段：MG / 生成画面 / B-roll / 全屏视觉 / PiP / 分屏 / 保持干净
- 官方结构参考：模板族 / 定制 / 无
- 导演判断：为什么这个位置值得做视觉
- 画面设计：观众最终看到什么
- 人物处理：全屏 / PiP / 分屏 / 原口播 + MG / 保持原画面
- 位置与尺寸：依据安全区；不适用写“不适用”
- 动画节奏：入场、强调、停留、退场
- ChatCut 用户提示词：短指令
- 导演约束层：布局、镜头、材质、节奏、可读性、保护区和禁止项
- 生成画面 prompt：仅需要时填写，否则写“不适用”
- 画面保护：脸、字幕、手势、产品、Logo、已有文字和运动路径
- 廉价风险：本 Beat 最可能做差的地方
- 必要性：1-5
- 高级感：1-5
- 信息增益：1-5
- 执行风险：低 / 中 / 高
- 质量结论：执行 / 降级为轻量 MG / 改成生成画面 / 改成真实素材 / 跳过 / 需要人工确认
```

保持干净的片段不必伪装成完整特效 Beat，但必须列出锚点和原因。

## 代表性 Beat

选择最能验证主视觉语言、文字层级、人物构图和动画节奏的中等风险 Beat。说明它验证什么、为什么不是最复杂的 Beat，以及通过后哪些 Beat 可以复用同一系统。

## 高风险确认项

单独列出：

- 会触发视频、图片或 MG 生成额度的点
- 可能误导真实产品、数据、地点或事件的点
- 需要人物参考、品牌素材或版权确认的点
- 可能因画面拥挤而改变人物构图的点

## 零 Beat 情况

如果所有候选都未通过 Quality Gate，明确写：本轮建议保持干净，并说明人物表达、画面拥挤、信息密度或事实风险。不要为了满足格式而保留低价值 Beat。
