# ChatCut 官方 Prompt 路由

## 核心原则

用户描述视觉任务后，先检索 `chatcut-official-catalog.md`，再决定复用级别。官方方案优先，但不绕过 CutDirector 的时间确认、人物处理、品牌素材、保护区、Quality Gate 和两次确认。

## 三档匹配

| 匹配级别 | 判断 | 执行 |
| --- | --- | --- |
| 完全匹配 | 信息任务、结构、媒介和主要动效机制均一致 | 优先调用官方入口或模板 ID；保留官方结构，只替换用户内容和必要的可编辑属性 |
| 部分匹配 | 信息任务一致，但人物位置、画幅、字幕、安全区或品牌风格冲突 | 复用官方信息结构和动效机制，重新布局；明确标记为“官方结构参考” |
| 无匹配 | 官方目录没有承担相同观看任务的条目 | 创建定制方案，不强套最相近模板 |

视觉风格相似但观看任务不同，不算匹配。不能因为颜色或材质接近，就用章节卡替代数据解释或用 Logo Reveal 替代口播图标提示。

## 路由步骤

1. 从用户原话提取：观看任务、准确内容、媒介、人物是否保留、画幅、时间锚点和主要动效。
2. 在完整目录中按任务词、模板名称和描述搜索候选项。
3. 最多保留三个候选，按“观看任务 > 信息结构 > 人物构图 > 动效机制 > 视觉风格”排序。
4. 记录选中项的官方名称、类别、模板/预设 ID、官方入口、最后核对日期和匹配级别。
5. 完全匹配时，优先通过 ChatCut Library 或官方入口定位同名模板；找不到可执行模板时降为部分匹配，不伪造调用结果。
6. 将用户文案、数据、品牌素材和时间点填入可编辑属性，不修改与需求无关的官方机制。
7. 检查真实画面的脸、字幕、手势、产品、Logo、文字和运动路径；冲突时允许镜像、重排或放弃模板。
8. 只执行一个代表性 Beat，验证实际开始、中间和结束帧，获得第二次确认后再扩展。

## 输出证据

Visual Beat Map 中增加：

```markdown
- 官方 Prompt 匹配：完全匹配 / 部分匹配 / 无匹配
- 官方名称：准确名称或“不适用”
- 模板/预设 ID：官方 ID 或“不适用”
- 官方入口：URL 或“不适用”
- 复用范围：保留的结构、动效和可编辑属性
- 定制范围：因人物、字幕、画幅、品牌或安全区而改变的部分
```

## 边界

- “官方”描述的是来源，不代表适合每条口播素材。
- 官方模板不可替代准确文案、数据和品牌验证。
- 不复制官方页面中的完整实现代码到 Skill；使用目录、官方入口和模板/预设 ID 定位。
- 官方目录过期或入口失效时重新同步，不根据记忆编造新名称。
