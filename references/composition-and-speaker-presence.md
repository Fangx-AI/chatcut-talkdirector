# 画面构图与人物露出

## 先决定人物是否有必要继续出现

生成画面、B-roll 或全屏视觉出现时，单独判断人物露出。不要默认右下 PiP，也不要默认完全切走人物。

| 模式 | 使用条件 | 不使用条件 |
| --- | --- | --- |
| 全屏视觉 + 原声 | 沉浸式概念、场景解释、章节转场、hook、结论或 1-3 秒强节拍 | 需要讲述者表情、信任、手势或持续教学 |
| PiP 人物小窗 | 较长补画面、教程、产品解释；人物存在能提升信任、理解或陪伴 | 片段过短、画面无安全区、主体需要完整构图 |
| 分屏 / 侧边人物 | 图表、产品、对比和操作与人物同等重要 | 强情绪、沉浸式视觉或空间过窄 |
| 原口播 + MG overlay | 关键词、数字、步骤、关系和轻量强调 | 原画面已拥挤或没有可读空间 |
| 保持干净 | 人物表达强、字幕拥挤、补画面会误导或削弱信任 | 有明确且高价值的视觉解释任务 |

## PiP 判断

- 短于 1.5 秒通常不使用 PiP，避免小窗闪现。
- 长于 3 秒且完全切走人物会显得像素材拼贴时，优先评估 PiP 或分屏。
- 生成画面需要沉浸、冲击或完整构图时，优先全屏。
- 人物手势、表情、身份或产品指向正在承担表达时，优先保留人物。
- PiP 会遮挡主体、文字、产品或运动路径时，改用分屏、全屏或原画面。

## 放置顺序

1. 标出字幕、Logo、人物脸、手势、产品、已有文字和主运动路径。
2. 找出真实空白区，不按固定角落猜测。
3. 评估左下、右下、右上、左上和侧边条。
4. 如果没有足够空间，选择分屏、全屏或保持原画面。

右下角只有在不遮挡字幕、主体、产品、文字和运动方向时才是候选。底部存在字幕时，右下通常不是安全区。

## PiP 尺寸和裁切

- 竖屏 PiP 通常占画面宽度的 22%-30%。
- 横屏 PiP 通常占画面宽度的 16%-24%。
- 需要缩得更小才能避让时，优先分屏或全屏。
- 保留脸、肩部和基本表情，不裁掉嘴、下巴或关键手势。
- 边框、圆角、阴影和描边服从主视觉系统，不默认玻璃卡或厚重发光框。
- 入场和退场与 Beat 同步，不像临时贴上的监控窗口。

## 输出

每个生成画面、B-roll 或全屏 Beat 必须写：

```markdown
- 人物处理：全屏视觉 / PiP / 分屏 / 原口播 + MG / 保持干净
- 位置与尺寸：依据实际安全区；不适用时写“不适用”
- 原因：人物存在或消失如何改善信任、理解、情绪或构图
- 保护区：字幕、Logo、脸、手势、产品、文字和运动路径
```
