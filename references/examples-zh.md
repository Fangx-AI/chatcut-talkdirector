# 中文示例

## 输入

60 秒竖屏知识口播，人物居中，底部双行字幕。核心结构是“误区 -> 真正问题 -> 三个步骤 -> 结论”。用户要求更高级、更有视觉节奏。

## 输出示例

# CutDirector 视觉方案

- 视频目标：把三步方法讲清楚，并让开场反转和结论留下记忆点
- 平台与画幅：短视频，9:16，60 秒
- 已知画面条件：人物居中；底部双行字幕；两侧空间有限
- 导演假设：没有品牌色，先使用克制的商业编辑风
- 主视觉语言：暖白和炭黑为主，一个亮绿色强调；粗体无衬线标题；细线和编号；短遮罩入场；无装饰纹理
- 建议视觉密度：4 个 Beat / 60 秒
- 全局保护区：底部 22% 字幕；人物脸和肩部；手势范围执行时确认
- 代表性 Beat：Beat 03，验证编号列表、人物构图、字幕避让和逐项动画

## Beat 01：误区

- 文案锚点：很多人以为效率低，是因为工具不够多
- 时间位置：约 00:00-00:05
- 视觉目的：建立 hook 和错误前提
- 推荐手段：轻量 MG
- 官方结构参考：Type emphasis 的信息强调结构，仅作参考，不套用完整模板
- 导演判断：只强调“工具不够多”，为下一句反转留出空间
- 画面设计：人物保持主画面；关键词在人物左侧短暂出现并被一条删除线划过
- 人物处理：原口播 + MG
- 位置与尺寸：左侧中上；底部字幕区净空
- 动画节奏：关键词快速入场，停留 1 秒；删除线在反转前完成；整体淡出
- 精确内容：工具不够多
- 可编辑字段：文字、强调色、字体、时长和位置
- 媒体/人物窗口：不适用（保留原画面）
- 参考素材职责：无
- 合成与背景：透明 alpha MG 叠加原画面；退场后恢复干净人物画面
- 声音处理：保留原口播；不生成新对白；不新增音效或音乐
- ChatCut 用户提示词：给开场增加一个克制的关键词误区标记，只突出“工具不够多”，随后划掉。
- 导演约束层：暖白、炭黑和亮绿；一次一个焦点；不震动、不故障、不使用粒子；保护人物脸和底部字幕。
- 生成画面 prompt：不适用
- 画面保护：脸、字幕和手势
- 廉价风险：删除线过粗或关键词过大会抢走人物
- 必要性：4
- 高级感：4
- 信息增益：3
- 执行风险：低
- 质量结论：执行

## Beat 02：真正的问题

- 文案锚点：真正的问题，是你的工作流没有入口
- 时间位置：约 00:05-00:13
- 视觉目的：强化反转和核心观点
- 推荐手段：全屏视觉
- 官方结构参考：Story Quote Card 的逐行落点逻辑，仅作结构参考
- 导演判断：这是全片核心句，短暂切全屏比继续叠 overlay 更清楚
- 画面设计：两行大字；“真正的问题”先出现，“工作流没有入口”以亮绿色落点
- 人物处理：全屏视觉 2 秒，只保留原声；随后回到人物
- 位置与尺寸：不适用
- 动画节奏：第一行上移进入，第二行延迟进入；完成后向两侧短退场
- 精确内容：真正的问题 / 工作流没有入口
- 可编辑字段：两行文字、亮绿色、字体、字号层级、时长和退场方向
- 媒体/人物窗口：不适用（短暂全屏）
- 参考素材职责：无
- 合成与背景：2 秒全屏不透明金句卡；退场后回到原人物画面
- 声音处理：全屏期间保留原口播；不生成新对白；不新增环境声
- ChatCut 用户提示词：创建一个 2 秒全屏反转金句卡，逐行显示“真正的问题 / 工作流没有入口”。
- 导演约束层：大留白；不使用玻璃卡和辉光；跟随原声重音；退场后无缝回到人物。
- 生成画面 prompt：不适用
- 画面保护：文字不得进入底部字幕习惯区
- 廉价风险：两行同时放大或持续发光
- 必要性：5
- 高级感：4
- 信息增益：4
- 执行风险：低
- 质量结论：执行

## Beat 03：三步系统

- 文案锚点：第一步 / 第二步 / 第三步
- 时间位置：约 00:13-00:51
- 视觉目的：解释和记忆
- 推荐手段：MG + 人物构图
- 官方结构参考：AI Numbered Talking-Head Overlay 的人物加列表结构，仅作参考
- 导演判断：三个步骤需要统一结构，并按口播逐项出现
- 画面设计：人物保持中央但缩到上半区可用范围；三项根据真实空白区在侧边和上方逐项出现
- 人物处理：原口播 + MG；执行前用真实帧决定列表左右侧，不使用 PiP
- 位置与尺寸：字幕区上方；不遮脸和手势
- 动画节奏：数字先落位，文字随后 4 帧进入；旧项降为次级但不消失；第三项后整体退场
- 精确内容：1 记录重复任务；2 保留能触发动作的信息；3 汇入同一个系统
- 可编辑字段：三项文字、颜色、字体、行距、时长、位置和人物窗口
- 媒体/人物窗口：人物窗口大小与位置可编辑；执行时按真实安全区确定
- 参考素材职责：无
- 合成与背景：透明 alpha MG 与原人物画面合成；整体退场后恢复全幅人物
- 声音处理：保留原口播；不生成新对白；编号词点不添加竞争音效
- ChatCut 用户提示词：创建一个三步口播编号 overlay，按原声逐项显示“记录重复任务 / 保留能触发动作的信息 / 汇入同一个系统”。
- 导演约束层：根据真实空白区决定列表侧；底部 22% 净空；文字、颜色、字体、时长和位置可编辑；无玻璃卡、粒子和持续辉光。
- 生成画面 prompt：不适用
- 画面保护：脸、肩、手势和底部字幕
- 廉价风险：三项一次性弹出，或旧项仍保持同等高亮
- 必要性：5
- 高级感：5
- 信息增益：5
- 执行风险：中
- 质量结论：执行；先做代表性 Beat

## Beat 04：结论

- 文案锚点：好的系统会减少你反复做决定
- 时间位置：约 00:51-01:00
- 视觉目的：记忆和收束
- 推荐手段：轻量 MG
- 官方结构参考：Talking-Head Keyword Card 的人物与关键词分区逻辑，仅作参考
- 导演判断：结论需要落点，但人物表情应继续承担信任
- 画面设计：保留人物，只让“减少反复做决定”在真实空白区出现一次
- 人物处理：原口播 + MG
- 位置与尺寸：执行时按真实空白区决定
- 动画节奏：短入场，停留到句末，随画面淡出
- 精确内容：减少反复做决定
- 可编辑字段：文字、强调色、字体、时长和位置
- 媒体/人物窗口：不适用（保留原画面）
- 参考素材职责：无
- 合成与背景：透明 alpha 关键词叠加原画面；句末随画面淡出
- 声音处理：保留原口播；不生成新对白；不新增音效或音乐
- ChatCut 用户提示词：在结论句加入一次克制的关键词强调：“减少反复做决定”。
- 导演约束层：使用同一亮绿强调色和同一遮罩入场；不新增图标或背景卡。
- 生成画面 prompt：不适用
- 画面保护：脸、字幕和手势
- 廉价风险：重复开场的大字冲击会让结尾过重
- 必要性：4
- 高级感：5
- 信息增益：3
- 执行风险：低
- 质量结论：执行

## 保持干净的片段

- 文案锚点：工具不会替你思考
- 原因：人物语气和表情承担真实感；先保持一拍干净，再出现结论关键词

## 代表性 Beat

先执行 Beat 03。它能一次验证字体、色板、编号层级、人物构图、字幕避让和逐项节奏；它是中等风险，不是最复杂或最昂贵的 Beat。通过后，Beat 01 和 Beat 04 可以复用同一文字系统。

## 高风险确认项

- Beat 03 需要查看真实手势范围后决定列表左右位置。
- 当前方案不需要视频或图片生成额度；MG 创建仍可能触发 ChatCut 确认。

## 第一次确认范围

- 主视觉语言：暖白、炭黑、单一亮绿色、粗体无衬线、细线编号和短遮罩入场。
- 人物处理：Beat 01、03、04 保留原人物；Beat 02 短暂全屏 2 秒；Beat 03 的人物窗口需按真实手势范围确认。
- 额度动作：只授权先创建和放置代表性 Beat 03 的 MG；Beat 01、02、04 以及任何其他时间线改动尚未授权。

## 用户确认后的执行顺序

只有用户同时确认上述主视觉语言、人物处理和 Beat 03 的 MG 额度动作后，才创建和放置 Beat 03。检查开始、中间和结束帧并展示实际结果；再次获得明确确认后，才扩展其他 Beat。
