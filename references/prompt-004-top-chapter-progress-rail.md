# Prompt 004: 顶部章节导航进度条

在视频顶部常驻一条低占用的信息条，同时完成两件事：用分段标签告诉观众“现在讲到哪一章”，用一条连续细线告诉观众“整条视频还剩多少”。适合日报、周报、课程、访谈、产品发布和其他具有明确章节边界的横屏内容。

[观看已验证演示视频](../assets/verified-prompts/prompt-004-top-chapter-progress-rail.mp4)

## 用户直接使用的 Prompt

```text
给 [整条视频或确认时间段] 的顶部添加一条常驻“章节导航 + 全片进度”信息条。

先根据逐字稿、时间线和已经确认的内容结构，建立按播放顺序排列的章节列表：[章节 1]、[章节 2]、[章节 3]……；片头和片尾存在时可单独保留。章节名称以中文短标签为主，不添加无意义的小号英文。每个章节格的宽度按照该章节在成片中的真实时长比例分配，短章节窄、长章节宽。

当前章节使用 [强调色] 和轻微深色底高亮，已经完成的章节保持可辨识但降低亮度，尚未开始的章节使用中性深色。章节切换必须对齐真实内容边界，不提前、不滞后、不闪烁。

在章节条底边增加一条 3-5 像素的全片进度线。进度线从整条视频第一帧的 0% 开始，按照全片播放时间匀速、连续地从左向右填满，到最后一帧达到 100%；跨章节时绝不清零、回跳或重新播放入场动画。进度线与当前章节高亮共享同一个时间基准。

默认用于横屏 16:9：章节条紧贴顶部安全区，高度约占画面高度的 4%-6%，文字在手机横屏预览中仍可快速读清。不得遮挡已有 Logo、标题、人物头部、产品界面或系统安全区；发生冲突时优先缩短章节名称、降低高度或改为“当前章节名称 + 连续进度线”，不要覆盖主体内容。

整体使用克制的深色底、细分隔线和单一强调色，不添加粒子、持续辉光、玻璃拟态、无关图标或多余英文。片头快速淡入或下落 6-10 帧后保持稳定，片尾在进度达到 100% 后与整条视频一起结束。

先展示第一帧、至少一个章节切换前后、中段进度和最后一帧，并提供一段实时播放预览让我确认；确认章节边界、标签、顶部安全区和进度连续性后，再扩展到整条视频。
```

## 观看任务

- 观众随时知道当前主题，不需要等待章节标题卡。
- 观众能预估剩余时长，连续进度线在跨章节时保持方向感。
- 章节标签负责内容定位，细进度线负责时间反馈；二者不能互相替代。
- 进度条是导航层，不抢正文画面、字幕或口播。

## 输入与时间确认

执行前必须确认：

1. 整条成片或目标时间段的准确起止时间。
2. 按播放顺序排列的章节名称，以及每一章的真实起止时间。
3. 片头、片尾是否作为独立章节。
4. 顶部已有 Logo、标题、系统 UI、人物头部或产品控件的位置。
5. 唯一强调色；未指定时从已确认的项目视觉语言中取一个高对比颜色。

不得仅凭素材文件名、镜头数量或自动摘要猜测章节边界。逐字稿、时间线标记和用户确认不一致时，以用户最后确认的边界为准。

## 信息结构与尺寸

- 章节格总宽度覆盖可用顶部安全区；每格宽度等于该章时长占目标总时长的比例。
- 标签优先控制在 2-6 个汉字。空间不足时先压缩文案，不把字号缩到手机端不可读。
- 片头和片尾可以只显示“片头”“片尾”，且按真实短时长获得较窄宽度。
- 当前章节可使用深色强调底与高亮文字；已完成和未开始章节必须仍有清楚的明度差。
- 全片进度线独立覆盖整条可用宽度，不按章节分段重置。
- 横屏 16:9 建议总高度为画面高度的 4%-6%，进度线为 3-5 像素；其他画幅按安全区等比例调整。

## 动画与同步

- 入场只发生一次：片头 6-10 帧淡入或轻微下落，然后贯穿全片。
- 章节高亮在真实章节边界切换，建议用 4-8 帧颜色交接，不横向滑过无关章节。
- 全片进度使用线性时间映射：`progress = (currentTime - startTime) / (endTime - startTime)`。
- 进度在任何章节边界前后都必须单调递增；禁止清零、回跳、停顿、追帧或重复入场。
- 最后一帧达到 100%，不得提前跑满；若片尾需要淡出，先完成进度再与整体同步退场。

## 安全区与降级

优先级从高到低：

1. 保护系统安全区、已有品牌 Logo、人物头部、字幕、标题和产品控件。
2. 缩短章节标签。
3. 降低条高，但保持移动端可读。
4. 把所有章节格降级为“当前章节名称 + 连续全片进度线”。
5. 顶部没有安全区时保持原画面，并请求用户确认新的位置；不得私自覆盖主体内容。

章节超过 8 个或单个标签过长时，不把所有文字挤进同一行。应合并语义相近章节、使用当前章节模式，或先请求用户确认更短标签。

## 可编辑属性

至少暴露：章节名称、章节边界、是否显示片头/片尾、条高、左右边距、字号、分隔线颜色、未开始颜色、已完成颜色、当前强调色、进度线高度、入场帧数和章节切换帧数。

章节宽度和进度值必须由真实时间自动计算，不要求用户填写像素宽度或关键帧百分比。

## 验证

至少检查并展示：

- 第一帧：进度为 0%，片头或首章状态正确，没有遮挡顶部主体。
- 第一个章节中段：全片进度与实际播放时间一致。
- 至少一个章节边界前后：标签高亮切换准确，细进度线连续且单调递增。
- 全片中段：章节宽度与真实时长比例一致，所有标签仍可读。
- 最后一帧：末章或片尾高亮正确，进度达到 100% 且不溢出。
- 实时播放预览：没有闪烁、清零、回跳、重复入场或字幕/人物冲突。

已验证案例使用 87.872 秒横屏成片，包含片头、五个主题章节和片尾；章节高亮按内容边界切换，底部细进度线以同一全片时间基准连续填满。
