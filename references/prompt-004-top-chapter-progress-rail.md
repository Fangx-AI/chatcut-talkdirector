# Prompt 004: 自适应章节导航与全片进度条

为各种题材、时长和画幅的视频提供持续的方向感：观众能知道“现在讲到哪里”和“整条视频还剩多少”。本 Prompt 不预设日报、口播、课程或产品视频，也不强制所有视频都显示完整章节列表；它先识别内容结构与真实安全区，再选择最合适的信息密度。

[观看已验证演示视频](../assets/verified-prompts/prompt-004-top-chapter-progress-rail.mp4)

## 用户直接使用的通用 Prompt

```text
分析 [整条视频或确认时间段] 的实际内容、画幅、节奏和安全区，为它添加一条自适应的“章节或段落导航 + 全片进度”信息条。不要套用固定题材、固定章节数量、固定画幅、固定位置或固定视觉风格。

先读取逐字稿、时间线、镜头边界、已有标题或章节标记，并检查所有代表画面中的人物、字幕、Logo、产品界面和平台安全区。只使用有事实依据并经过确认的章节或段落名称与真实起止时间，不根据素材文件名、自动摘要或镜头数量自行编造结构。

根据视频实际情况选择一种形态：
1. 完整章节导航：适合有清晰章节，且章节数量、标签长度和当前画幅允许舒展排布的视频。按照每章真实时长比例分配宽度，当前章高亮，已完成和未开始章节保持可区分；不把“最多几章”写成跨视频的固定限制。
2. 当前章节模式：适合章节较多、标签较长、竖屏或方形视频。只显示当前章节名称与全片进度，不把所有标签挤在一行。
3. 纯进度模式：适合没有明确章节、内容很短或画面信息密度很高的视频。只保留一条克制的全片进度线，不虚构章节。
4. 保持干净：没有稳定安全区、会遮挡关键信息，或进度提示不能增加理解时，不强行添加。

位置默认优先考虑顶部，但顶部不是硬性要求。根据实际画幅和全片安全区选择顶部、底部或其他安全边缘；横屏、竖屏、方形、屏幕录制、游戏、访谈和人物口播分别重新检查，不能沿用固定坐标。任何位置都不得遮挡脸、字幕、手势、Logo、标题、产品控件、代码、图表或平台 UI。

无论使用哪种可见形态，都让全片进度使用同一个真实时间基准：从目标范围第一帧的 0% 开始，按照播放时间连续、单调地填充，到最后一帧达到 100%。跨章节或跨镜头时绝不清零、回跳、停顿、提前跑满或重新播放入场动画。章节高亮与进度线必须共享同一组真实时间边界。

视觉风格继承当前视频，而不是固定使用深色科技风。字体、语言、颜色、圆角、线条、透明度和动效强度应与原片一致；标签沿用原视频主要语言，文字短而可读，不额外添加无意义英文、图标、粒子、持续辉光或装饰性动画。尺寸根据画幅与观看设备自适应，优先保证手机预览可读和正文画面完整。

整条导航只入场一次并保持稳定。章节切换使用克制的颜色、字重或底色交接，不用大幅横向滑动抢夺注意力；最后一帧进度达到 100% 后再与视频同步结束。

先提交结构识别结果、所选形态、章节或段落表、位置与安全区依据，然后展示第一帧、至少一个切换边界前后、中段和最后一帧，并提供实时播放预览让我确认。确认后再扩展到整条视频。
```

## 核心判断

这个 Prompt 首先判断“是否值得显示章节”，其次才决定怎么画：

| 视频实际情况 | 默认形态 |
| --- | --- |
| 清晰章节数量与标签长度适合当前画幅 | 完整章节导航 + 全片进度 |
| 章节多、标签长、竖屏或方形 | 当前章节名称 + 全片进度 |
| 无明确章节、短视频或高信息密度画面 | 纯全片进度 |
| 没有稳定安全区或没有观看价值 | 保持干净 |

形态是导演判断，不是让用户填写的参数。用户只需确认内容结构、视觉方向与代表画面。

## 输入与事实确认

执行前确认：

1. 目标范围的准确起止时间与总时长。
2. 视频是否存在真实、值得显示的章节或段落。
3. 若存在章节：按播放顺序排列的原文短标签与真实起止时间。
4. 画幅、发布平台和全片代表帧中的安全区。
5. 原片主要语言、字体、颜色和动效强度。
6. 已有字幕、标题、Logo、人物、产品 UI、代码、图表和系统控件。

逐字稿、时间线标记和用户确认不一致时，以用户最后确认的边界为准。没有章节事实时使用纯进度或保持干净，不补写“片头、第一章、总结”等虚构结构。

## 自适应布局

### 完整章节导航

- 仅在标签数量、长度和可用空间允许时使用；数量阈值由实际画幅和可读性决定。
- 每格宽度等于该章时长占目标总时长的比例。
- 若短章节的比例宽度不足以容纳文字，切换到当前章节模式；不得为了排版改成虚假的等宽章节。
- 当前、已完成和未开始状态应有明确但克制的层级差。

### 当前章节模式

- 只显示当前章节名称，可配合前后章节的极简方向提示，但不堆满全部标签。
- 章节名称按真实边界替换，全片进度连续运行。
- 适合竖屏、方形、章节较多、长标签或顶部横向空间不足的内容。

### 纯进度模式

- 不显示章节名称，只提供低占用的时间反馈。
- 适合没有可验证章节、时长很短、镜头切换密集或原画面已经承担全部解释任务的视频。
- 进度线仍然必须从 0% 连续到 100%。

### 位置与尺寸

- 顶部优先，但必须基于真实安全区；底部已有字幕或播放器控件时不能机械下移。
- 横屏通常能容纳完整章节；竖屏和方形通常优先当前章节或纯进度，但最终以实际标签和安全区为准。
- 屏幕录制要避开浏览器标签栏、应用导航和系统状态栏。
- 人物视频要保护头顶、眼睛、手势与姓名条。
- 游戏、体育、演示和产品 UI 要保护比分、地图、菜单、代码与关键控件。
- 字号、条高、边距和线宽按照画幅、分辨率、帧率与发布平台自适应，不写死像素值。

## 时间与动画

- 入场只发生一次，时长按帧率换算为约 0.2-0.35 秒的轻淡入或短位移。
- 全片进度使用线性时间映射：`progress = (currentTime - startTime) / (endTime - startTime)`。
- 章节切换严格对齐已确认边界，并与全片进度使用同一时钟。
- 进度在任意章节、镜头或素材边界前后都必须单调递增。
- 最后一帧达到 100%；若需要退场，先完成进度再同步结束。
- 不要求所有题材使用相同切换动画；只要求切换克制、可读且不打断主体。

## 视觉适配

- 继承原片的语言、字体、色彩、圆角、线条与运动气质。
- 新闻、科技、生活、教育、商业、游戏和纪录片应呈现不同的视觉语气。
- 标签沿用原视频主要语言；品牌或专有名词需要保留原文时可例外。
- 强调色优先来自已有品牌或画面色，不凭空加入第二套视觉系统。
- 保持高对比可读，但不默认深色底、霓虹色、科技网格、玻璃拟态或英文标签。

## 降级与阻断

依次尝试：

1. 缩短标签但保留原意。
2. 从完整章节导航切换为当前章节模式。
3. 从当前章节模式切换为纯进度。
4. 改用经过确认的其他安全边缘。
5. 仍然会遮挡或没有观看价值时保持干净。

不得用缩小到不可读、覆盖字幕、改变章节真实时长、假造等宽章节或隐藏关键画面来“保住”完整导航。

## 可编辑属性

至少暴露：显示形态、章节名称、章节边界、位置、条高、边距、字号、字体、当前/已完成/未开始状态、强调色、背景透明度、进度线高度、入场时长和章节切换时长。

章节宽度和进度值由真实时间自动计算，不要求用户填写像素宽度、百分比关键帧或固定坐标。

## 验证

至少检查并展示：

- 结构：所选形态确实适合章节数量、标签长度、画幅和安全区。
- 第一帧：进度为 0%，没有遮挡关键内容。
- 中段：进度与真实播放时间一致，文字在目标设备上可读。
- 边界：至少一个章节或段落切换准确，进度连续且单调递增。
- 多样帧：不同镜头、人物位置、字幕状态和产品 UI 下均保持安全。
- 最后一帧：进度达到 100%，不溢出、不提前结束。
- 实时播放：没有闪烁、清零、回跳、重复入场或对主体内容的持续干扰。

当前已验证案例是 87.872 秒横屏成片，包含片头、五个主题段落和片尾。它只用于证明“章节切换与连续全片进度共用同一时间基准”；题材、画幅、标签数量、位置和视觉风格不是通用 Prompt 的固定模板。
