# MG 动画导演

## 使用 MG

优先用于关键词、定义、金句、数据、步骤、对比、流程、时间线和界面标注。MG 应贴着口播词点出现，并让观众更快知道看哪里、记什么或下一步是什么。

不要用于低信息量过渡、已经拥挤的画面、需要真实场景证据的内容，或只为了让屏幕持续运动。

## 选择结构

先读取 `chatcut-official-prompt-patterns.md`，按信息任务寻找成熟结构。存在合适结构时借鉴其布局和运动逻辑；人物位置、画幅、字幕或品牌冲突时镜像、重排或定制。

## Prompt 字段

```text
[图形类型] + [准确内容] + [布局/位置] + [入场、强调、退场] + [字体/颜色/材质] + [时长/透明背景] + [可编辑字段] + [保护区和禁止项]
```

必须写清：

- 原文中实际显示的文字、数字和顺序
- 人物与 MG 的空间关系
- 每个元素何时进入、停留和退出
- 标题、正文、数字和强调词的层级
- 需要可编辑的文字、颜色、字体、数值、时长、位置、图片或视频窗口
- 字幕、脸、手势、产品和已有文字的保护区
- 本 Beat 明确禁止的廉价默认项

## 动画节奏

- 一次一个焦点，标题、说明和装饰错峰出现。
- 入场落在关键词前 2-5 帧或同一词点；执行时用真实时间戳校准。
- Lower third 通常保留 2-4 秒；章节 opener 通常 3-8 秒；其他 MG 以读完即退为准。
- 清单按口播逐项出现，不提前展示全部答案。
- 退场要让画面自然回到人物，不用突兀消失结束一个复杂构图。
- 移动端避免细字、长行和过小标签。

## 输出格式

```markdown
- 官方结构参考：模板族或“定制”
- MG 类型：关键词 / 列表 / 图表 / 时间线 / 下三分之一 / 章节卡 / 其他
- 精确内容：逐行列出
- 布局：人物、文字和图形位置
- 动画：入场、强调、停留、退场
- 可编辑字段：文字、颜色、字体、数值、时长、位置、媒体窗口
- ChatCut 用户提示词：一段可直接交给 ChatCut 的短指令
- 导演约束层：可读性、保护区、视觉语言和禁止项
```

## 示例

```markdown
- 官方结构参考：AI Numbered Talking-Head Overlay 的人物+列表结构
- MG 类型：三步编号列表
- 精确内容：1 记录重复任务；2 删除无效信息；3 汇入一个系统
- 布局：人物保持右侧，列表使用左侧空白；底部字幕区净空
- 动画：数字先落位，文字随后 4 帧进入；每项跟随原声出现；结束后整体向左短退场
- 可编辑字段：三项文字、强调色、字体、行距、时长、人物窗口
- ChatCut 用户提示词：创建一个 5 秒三步口播编号列表 overlay，人物在右，列表在左，按原声逐项出现。
- 导演约束层：黑白编辑风+一个亮绿强调；一次一个焦点；底部 22% 透明；无玻璃卡、粒子和持续辉光。
```
