# Prompt 001: 手势锚定通用 Logo 弹出

## 用户直接使用的 Prompt

```text
在 [时间段]，给人物两侧的指向手势添加 [品牌 A] 和 [品牌 B] 的官方 Logo 弹出特效：[品牌 A] 在画面左侧，[品牌 B] 在画面右侧，分别跟随对应手指抬起时弹出，手势结束时退场。请自动获取可验证的官方 Logo，保持人物全屏，不遮挡脸、字幕、手和产品，并先展示关键帧让我确认。
```

品牌是动态输入，不限于已验证案例。ChatCut、Claude Code、Cursor、Hyperframes 或其他产品均使用同一条用户 Prompt；不得要求用户理解或填写裁切、缩放、SVG viewBox 等内部参数。

已验证案例：

```text
在 00:00.75-00:03.20，给人物两侧的指向手势添加 ChatCut 和 OpenAI 的官方 Logo 弹出特效：ChatCut 在画面左侧，OpenAI 在画面右侧，分别跟随对应手指抬起时弹出，手势结束时退场。请自动获取可验证的官方 Logo，保持人物全屏，不遮挡脸、字幕、手和产品，并先展示关键帧让我确认。
```

## 时间确认门

1. 在获取 Logo、创建 MG 或修改时间线前，必须获得用户确认的准确时间段。
2. 用户未提供时间段时，只检查候选片段并询问一个问题：`请确认要添加特效的准确时间段。`
3. 同一片段存在多个手势时，展示候选代表帧；不得仅凭“手在动”自动选择触发动作。
4. 确认时间段后，检查起始、每次手势抬起、双手稳定和手势收回的真实帧。

## 官方 Logo 获取

按以下顺序获取，并保留来源：

1. 项目内已有的官方素材
2. 官方 Brand、Press 或 Media Kit
3. 官方网站提供的 SVG 或 PNG
4. 官方应用商店图标
5. 官方网站 favicon
6. 无法验证时询问用户，不使用相似图替代

真实品牌 Logo 不得用图片生成或视频生成模型仿制。官方素材是组合规范图时，可以无损裁取其中的单体标志；不得重绘、改变比例或添加品牌未授权的颜色和纹理。

## 动态品牌解析

1. 将每个品牌视为独立输入，不复用另一个品牌的裁切、缩放或颜色参数。
2. 先确认准确产品身份。区分公司、产品、应用和功能名称，例如 `Claude` 与 `Claude Code`；同名或来源不明确时，只询问一个澄清问题。
3. 为每个品牌记录：规范名称、官方来源 URL、素材类型、深浅色版本、是否为单体标志、透明留白和可用背景。
4. 获取素材后先检查实际像素或 SVG 画布。不得只根据文件名判断它是单体 Logo。
5. 组合规范图只裁取目标单体标志。裁切参数属于该素材实例，不得写死为“右侧 Logo”或某个品牌的全局默认值。
6. 普通单体 Logo 默认使用 `contain`；存在异常留白时，使用独立的 `zoom`、`offsetX`、`offsetY` 或规范化后的派生素材。
7. 深色 Logo 使用浅色卡片，浅色 Logo 使用品牌允许的深色或中性卡片；不得通过描边或滤镜擅自改造 Logo。
8. 无法获得可验证官方素材时，停止该品牌的执行并询问用户，不影响已经确认的其他品牌。

## 动态输入与布局

- 一个品牌：锚定唯一明确手势；没有明确手势时使用真实安全区，不假造第二个图标。
- 两个品牌：分别映射左右或先后手势；每个图标独立触发。
- 超过两个品牌：优先按口播或手势顺序逐个出现。若同时出现会拥挤，先提交降密度方案，不把多个图标堆满人物周围。
- 用户没有指定左右关系时，根据手势、空白区和口播顺序提出映射并等待确认。

## 导演与动效规则

- 保留原口播人物全屏，以透明 MG 叠加图标；不创建人物 PiP。
- 每个图标锚定对应指尖上方的真实安全区，不固定套用左右角落。
- 横屏卡片边长建议为画面宽度的 12%-16%；圆角约为卡片边长的 20%-24%。
- 使用暖白或中性白卡片、细描边和柔和阴影；Logo 保持原始比例。
- 每个图标独立触发。入场使用淡入、轻微上移和 `0.82 -> 1.08 -> 1.00` 的缩放回弹。
- 入场约 10-14 帧；触发点落在手指抬起至稳定前约 2-5 帧，不抢跑到手势之前。
- 两个手势先后出现时，图标必须按真实顺序错峰进入。
- 手势收回时，图标在约 8-12 帧内缩小至约 `0.92` 并淡出。
- 保护脸、眼睛、字幕、手指、产品、已有 Logo、文字和主要运动路径。
- 不添加粒子、持续辉光、镜头抖动、玻璃拟态或无关装饰。

## 可编辑属性

每个图标至少独立暴露：素材、位置、Logo 缩放、裁切模式、水平偏移、垂直偏移、卡片颜色和触发延迟。公共属性至少暴露：卡片尺寸、圆角、阴影强度、上移距离、起始缩放、回弹缩放和退场时间。

不得使用只适配已验证案例的固定实现，例如把 OpenAI 组合素材的裁切参数硬编码到“右侧图标”。已验证案例可以保留专属参数，但新品牌必须重新检查和计算。

## 验证

执行后至少检查：

- 每个官方素材在卡片中的独立预览
- 特效前一帧
- 左侧图标入场与稳定帧
- 右侧图标入场与稳定帧
- 双图标共同停留帧
- 退场开始、接近消失和完全消失帧

向用户展示实际结果并获得第二次确认后，才能把该视觉系统扩展到其他 Beat。
