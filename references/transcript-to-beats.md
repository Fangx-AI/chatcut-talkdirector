# 从 Transcript 提取视觉节拍

## 分段

按语义和呼吸点切分，不按固定字数切分。每段保留：

- `anchor`：原文中最短且唯一的锚点
- `time`：精确时间或“约”时间
- `type`：hook / 观点 / 步骤 / 数据 / 对比 / 案例 / 场景 / 反转 / 结论 / CTA / 过渡
- `purpose`：解释 / 记忆 / 节奏 / 情绪 / 转场
- `candidate_medium`：MG / 生成画面 / B-roll / 全屏 / PiP / 分屏 / 保持干净
- `reason`：为什么视觉比原画面更好

不要改写用户观点来制造更强的视觉点。执行用文字必须回到原文核对。

## 候选规则

| 文案信号 | 优先考虑 | 默认避免 |
| --- | --- | --- |
| “真正的问题是”“但是”“相反” | 反转大字、前后对比、短全屏 Beat | 同时叠加多种故障和震动 |
| “第一、第二、第三” | 编号列表、时间线、检查清单 | 每一步使用不同模板 |
| 百分比、金额、增长、排名 | 数字强调、图表、对比 | 没有数据来源时伪造完整图表 |
| 抽象关系或隐喻 | 生成画面、简图、流程动画 | 与文案无关的氛围 B-roll |
| 产品动作或操作 | 产品录屏、界面标注、分屏 | 生成虚假产品界面冒充真实功能 |
| 人物故事或真实情绪 | 保持人物、轻量字幕强调 | 长时间切走人物 |
| 章节开始或结论 | 章节卡、全屏金句、短呼吸点 | 没有退场逻辑的长标题卡 |
| 普通过渡句 | 保持干净 | 为了密度硬加特效 |

## 候选优先级

为候选点计算导演优先级，不把这个分数展示成绝对真理：

- 结构价值：0-2，是否承担 hook、反转、步骤、结论或 CTA。
- 信息价值：0-2，是否包含需要看见的数字、关系、流程或定义。
- 记忆价值：0-2，是否是观众应带走的核心句。
- 视觉风险：0 到 -2，是否拥挤、难以准确生成、会遮挡或风格跳脱。

总分 3 以上进入候选；低于 3 默认保持干净。候选仍必须通过 Quality Gate。

## 时间锚点

- 有词级时间戳时，让入场落在关键词前 2-5 帧或同一词点，不能明显晚到。
- 只有句级时间戳时，使用句首、停顿或语义重音估计，并标注需要执行时校准。
- 没有时间戳时只写原文锚点和相对顺序，不编造 `00:00:00`。
- Beat 开始前保留必要的视觉准备，结束后给人物画面一个干净恢复点。

## 密度裁剪

候选过多时按以下顺序删除：

1. 只承担“动一下”的 Beat。
2. 与前一个 Beat 使用相同信息结构的 Beat。
3. 会遮挡人物、字幕或产品的 Beat。
4. 信息增益低于人物表达的 Beat。

保留的 Beat 应覆盖不同工作，而不是平均分布特效。
